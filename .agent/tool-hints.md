# Подсказки по инструментам и подходам

Этот файл помогает развивать инструкции `instructions/agent-start.ru.md` и `instructions/agent-start.en.md`: агент должен не только выполнять задачу, но и вовремя подсказывать пользователю полезные способы работы.

## Когда подсказывать

- Пользователь выполняет повторяемое действие вручную.
- Видно, что задачу можно ускорить скриптом, шаблоном, тестом, линтером, хуком или CI-проверкой.
- Появился риск потери контекста между агентами или сессиями.
- Проект начинает захламляться временными файлами.
- Требуется схема, документ, таблица или другой артефакт с понятным местом хранения.

## Что предлагать

- Проектный файл правил агента, подходящий для текущей среды.
- Файл истории изменений с понятным форматом записей.
- Контекстный файл с исходными вводными, допущениями и решениями.
- Небольшие шаблоны для повторяемых рабочих документов.
- `.drawio` для схем и контекстный Markdown-файл рядом.
- `rg` для быстрого поиска по проекту.
- Git-коммиты с осмысленными сообщениями после завершенных этапов.
- Локальные тесты или минимальную проверку перед финальным ответом.

## Как подсказывать

- Давать 1-3 конкретные подсказки, связанные с текущей задачей.
- Не превращать подсказки в длинную лекцию.
- Отделять факт от предложения: что уже сделано, что можно добавить дальше.
- Не навязывать пользователю фиксированную структуру каталогов, если проект уже имеет свои соглашения.
