# Требования и идеи развития

## Обязательные требования

- Главный пользовательский артефакт должен оставаться однофайловым для каждого языка.
- Проект должен поддерживать русский и английский языки.
- Русская и английская версии инструкции должны быть содержательно синхронизированы.
- Инструкция должна описывать действия агента, а не жестко задавать дерево каталогов.
- Агент должен сначала найти существующие правила проекта и дополнить их нашими правилами.
- Если правил нет, агент должен создать минимальную точку входа, подходящую для текущей среды.
- Корень проекта должен оставаться чистым.
- Агент должен вести историю изменений и фиксировать исходные вводные, допущения и решения.
- Агент должен писать пользователю дату, время и часовой пояс в каждом сообщении.
- Агент должен подсказывать полезные инструменты и подходы, когда это помогает работе.
- Для схем агент должен использовать `.drawio`, создавать рядом контекстный Markdown-файл и размещать элементы просторно, равномерно, без наложения блоков, стрелок и подписей.

## Практический вывод по формату

Markdown в 2026 году остается нормальным базовым форматом для таких инструкций:

- он читаем человеком без специальных инструментов;
- хорошо переносится между агентами;
- поддерживает заголовки, списки, кодовые блоки и ссылки;
- из него легко делать адаптеры под конкретные среды.

Но одного `.md` недостаточно как универсального автозагрузчика для всех агентов. Поэтому стратегия такая:

- держать мастер-инструкции в Markdown-файлах по языкам;
- просить агента встроить наши правила в уже существующий механизм проекта;
- для отдельных сред при необходимости добавить адаптеры, но не требовать их от каждого проекта.

## Структура публичных материалов

- `README.md`: двуязычная витрина проекта.
- `instructions/agent-start.ru.md`: русская однофайловая инструкция.
- `instructions/agent-start.en.md`: английская однофайловая инструкция.
- `LICENSE`: MIT License.
- `AGENTS.md`: правила для агентов, работающих над этим репозиторием.
