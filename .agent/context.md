# Контекст проекта

## Исходные вводные

- Дата создания каркаса: `2026-06-28 23:29:01 +07:00`.
- Пользователь хочет простое решение: один Markdown-файл с инструкцией для ИИ-агента, который может "обжить" каталог проекта.
- Минимальные ожидаемые правила:
  - в каждом сообщении указывать дату, время и часовой пояс;
  - вести историю изменений для быстрой передачи контекста человеку или другому агенту;
  - фиксировать исходные вводные, допущения и решения;
  - давать пользователю подсказки по инструментам и подходам;
  - не хранить планы, черновики, артефакты и временные файлы в корне проекта;
  - для схем использовать `.drawio` и контекстный Markdown-файл рядом.

## Допущения

- Двуязычность лучше реализовать двумя отдельными инструкциями: пользователь все равно передает агенту один файл, но выбирает язык.
- `README.md` должен быть двуязычной витриной с ссылками на обе инструкции.
- Русская и английская инструкции должны быть содержательно синхронизированы.
- MIT-лицензия подходит как практичный permissive default для открытого репозитория с текстовыми инструкциями и шаблонами.
- Корневой `AGENTS.md` используется только как инструкция для агентов, работающих над этим репозиторием.
- Служебная память разработки этого репозитория хранится в `.agent/`.
